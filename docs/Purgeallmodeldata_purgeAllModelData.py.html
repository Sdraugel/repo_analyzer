purgeAllModelData.py

Description:

This Python script is designed to purge all data for a specified model from EDEX, a data exchange system. Here's a breakdown of what the script does: 1. It begins by importing necessary modules and setting up a logger for debugging and information tracking purposes. 2. The `__initLogger` function is defined to initialize the logger. The logger's level is set to DEBUG, and a StreamHandler is added to the logger. The StreamHandler's level is set to INFO, and it's formatted to display the time, logger's name, log level, and the log message. 3. The `__parseCommandLine` function is defined to parse command-line arguments. The arguments include the host where the EDEX server is running, the port on which the EDEX server is listening, and the names of the models to be purged. The function ensures that there are no duplicate model names in the list. 4. The `main` function is defined to carry out the main operations. It initializes the logger, parses the command-line arguments, and creates a Thrift client with the provided host and port. For each model specified, it sends a request to the EDEX server to delete all data for that model. If the data is successfully deleted, it logs a success message. If an error occurs during the deletion, it logs an exception message. 5. Finally, the script checks if it is being run as the main module, and if so, it calls the `main` function to execute the script. This script is a good example of a utility script that can be run from the command line to perform a specific task, in this case, purging data from a server.
